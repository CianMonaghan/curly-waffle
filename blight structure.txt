
0. Preface
Definitions used in this document:
·       stats = string enum { STR, DEX, CON, WIS, INT, CHA }
·       Character is the canonical owner of all mutable state.
·       Attachments (race/background/class/subclass/features/items) never mutate Character state directly.
·       All changes must occur through Character modification methods, and must be reversible.
·       ComputeAllDerived(character) is executed after every apply/reverse operation, using only the stages described in the original specification.

2. Feature Mechanics (Technical)
This section explains how features stored on attachments affect the Character using modification methods, while remaining reversible.
·       Attachments store FeatureObjects. FeatureObjects are declarative; they do not mutate state.
·       When an event occurs (e.g., equip item), Character iterates the relevant features and calls applyFeature(trigger, sourceId, feature).
·       applyFeature(...) calls feature.buildModifications(context) to get a list of Modifications.
·       Character applies each Modification via applyModification(mod), which dispatches into canonical modification methods (alterStat, addToList, setMapValue, etc.).
·       Character stores an AppliedFeatureRecord containing AppliedModifications (including reversalData when necessary).
·       When the event is undone (unequip/detune/remove attachment), Character reverses exactly the previously-applied feature instances using reverseFeature(featureInstanceId).
·       After every apply/reverse operation Character runs ComputeAllDerived(this) using only the original pipeline stages.
2.1 Example: Feature → Modification → Character Mutation
Example feature: Ability Score Increase (+2 STR) applied when added.
// FeatureObject
const ASI_STR_PLUS_2 = {
  Id: 'feat-asi-str-2',
  Name: 'Ability Score Increase (STR +2)',
  Description: 'Increase STR by 2.',
  Trigger: 'onAdd',
  buildModifications(ctx) {
	return [{ Type: 'statDelta', Payload: { stat: 'STR', delta: 2 } }];
  }
};
 
// On class/race/feature attach:
character.applyFeature('onAdd', sourceId, ASI_STR_PLUS_2);
2.2 Example: Hard-Set Override (requires captured reversal value)
Example: item sets STR to 19 while attuned; reversal restores previous value captured during apply.
const OGRE_POWER = {
  Id: 'feat-ogre-power',
  Name: 'Gauntlets of Ogre Power',
  Description: 'Your STR score becomes 19 while attuned.',
  Trigger: 'onAttune',
  buildModifications(ctx) {
	return [{ Type: 'statSet', Payload: { stat: 'STR', value: 19 } }];
  }
};
 
3. Full Method Reference
Each method is listed with: signature, description, implementation details (JS pseudocode), and returns/end result.
 
3.1 Character — Feature Application & Modification Dispatch
Method: applyFeature(trigger: string, sourceId: string, feature: FeatureObject, context?: object): string
Description:
Builds and applies the feature’s modifications exactly once. Generates a unique featureInstanceId so identical features can coexist across multiple sources. Stores an AppliedFeatureRecord for precise reversal.
Implementation details (JavaScript pseudocode):
applyFeature(trigger, sourceId, feature, context = {}) {
  const featureInstanceId = `${sourceId}:${feature.Id}:${trigger}:${this._randomId()}`;
  const mods = feature.buildModifications({ character: this, sourceId, trigger, ...context }) || [];
  const applied = [];
 
  for (const mod of mods) {
	applied.push(this.applyModification(mod));
  }
 
  this.ActiveFeatureRefs.set(featureInstanceId, {
	featureInstanceId,
	sourceId,
	featureId: feature.Id,
	trigger,
	applied,
  });
 
  this.recomputeAllDerived();
  return featureInstanceId;
}
Returns / End result:
Returns featureInstanceId. Character is mutated by the feature once; derived fields are recomputed.
 
Method: reverseFeature(featureInstanceId: string): void
Description:
Reverses a previously applied feature instance by undoing its applied modifications in reverse order. Deletes its record and recomputes derived values.
Implementation details (JavaScript pseudocode):
reverseFeature(featureInstanceId) {
  const rec = this.ActiveFeatureRefs.get(featureInstanceId);
  if (!rec) return;
 
  for (let i = rec.applied.length - 1; i >= 0; i--) {
    this.reverseAppliedModification(rec.applied[i]);
  }
 
  this.ActiveFeatureRefs.delete(featureInstanceId);
  this.recomputeAllDerived();
}
Returns / End result:
Returns void. Restores state affected by that feature instance.
 
Method: applyModification(mod: Modification): AppliedModification
Description:
Applies one Modification by dispatching into canonical mutation methods. Captures reversalData for operations that need it (set/map).
Implementation details (JavaScript pseudocode):
applyModification(mod) {
  switch (mod.Type) {
	case 'numberDelta':
  	this.alterNumberField(mod.Path, mod.Payload.delta);
  	return { modification: mod };
 
	case 'numberSet': {
  	const prev = this.getNumberField(mod.Path);
  	this.setNumberField(mod.Path, mod.Payload.value);
  	return { modification: mod, reversalData: { prev } };
	}
 
	case 'statDelta':
  	this.alterStat(mod.Payload.stat, mod.Payload.delta);
  	return { modification: mod };
 
	case 'statSet': {
  	const prev = this.BaseStats[mod.Payload.stat];
  	this.setStat(mod.Payload.stat, mod.Payload.value);
  	return { modification: mod, reversalData: { prev } };
	}
 
	case 'listAdd':
  	this.addToList(mod.Path, mod.Payload.value);
  	return { modification: mod };
 
	case 'listRemove':
  	this.removeFromList(mod.Path, mod.Payload.value);
  	return { modification: mod };
 
	case 'mapSet': {
  	const hadKey = this.mapHasKey(mod.Path, mod.Payload.key);
  	const prev = hadKey ? this.getMapValue(mod.Path, mod.Payload.key) : undefined;
  	this.setMapValue(mod.Path, mod.Payload.key, mod.Payload.value);
  	return { modification: mod, reversalData: { hadKey, prev } };
	}
 
	case 'mapDelete': {
  	const hadKey = this.mapHasKey(mod.Path, mod.Payload.key);
  	const prev = hadKey ? this.getMapValue(mod.Path, mod.Payload.key) : undefined;
  	this.deleteMapKey(mod.Path, mod.Payload.key);
  	return { modification: mod, reversalData: { hadKey, prev } };
	}
 
	case 'weaponRegister':
      this.registerEquippedWeapon(mod.Payload.weaponId);
  	return { modification: mod };
 
	case 'weaponUnregister':
      this.unregisterEquippedWeapon(mod.Payload.weaponId);
  	return { modification: mod };
 
	default:
  	throw new Error(`Unknown modification type: ${mod.Type}`);
  }
}
Returns / End result:
Returns an AppliedModification containing the original modification and any captured reversalData.
 
Method: reverseAppliedModification(applied: AppliedModification): void
Description:
Reverses one AppliedModification. Uses captured reversalData for set/map operations; uses inverse operations for delta operations.
Implementation details (JavaScript pseudocode):
reverseAppliedModification(applied) {
  const mod = applied.modification;
  switch (mod.Type) {
	case 'numberDelta':
  	this.alterNumberField(mod.Path, -mod.Payload.delta);
  	return;
	case 'numberSet':
  	this.setNumberField(mod.Path, applied.reversalData.prev);
  	return;
	case 'statDelta':
  	this.alterStat(mod.Payload.stat, -mod.Payload.delta);
  	return;
	case 'statSet':
  	this.setStat(mod.Payload.stat, applied.reversalData.prev);
  	return;
	case 'listAdd':
  	this.removeFromList(mod.Path, mod.Payload.value);
  	return;
	case 'listRemove':
  	this.addToList(mod.Path, mod.Payload.value);
  	return;
	case 'mapSet':
  	if (applied.reversalData.hadKey) this.setMapValue(mod.Path, mod.Payload.key, applied.reversalData.prev);
  	else this.deleteMapKey(mod.Path, mod.Payload.key);
  	return;
	case 'mapDelete':
  	if (applied.reversalData.hadKey) this.setMapValue(mod.Path, mod.Payload.key, applied.reversalData.prev);
  	return;
	case 'weaponRegister':
      this.unregisterEquippedWeapon(mod.Payload.weaponId);
  	return;
	case 'weaponUnregister':
      this.registerEquippedWeapon(mod.Payload.weaponId);
  	return;
	default:
  	throw new Error(`Unknown modification type: ${mod.Type}`);
  }
}
Returns / End result:
Returns void. The modification’s effect is undone.
 
Method: _randomId(): string
Description:
Generates a unique id segment to avoid collisions between feature instances. Any stable implementation is acceptable.
Implementation details (JavaScript pseudocode):
_randomId() {
  return Math.random().toString(16).slice(2) + Date.now().toString(16);
}
Returns / End result:
Returns a string.

alterStat(stat, delta) {
  if (!this.isValidStat(stat)) throw new Error('invalid stat');

  this.BaseStats[stat] = cur + delta;
}

3.3 Character — Generic Path/List/Map Utilities
Method: getPath(path: string): any
Description:
Resolves a dot-delimited path from the Character root. Used by generic modification types.
Implementation details (JavaScript pseudocode):
getPath(path) {
  const parts = path.split('.');
  let cur = this;
  for (const p of parts) {
	if (cur == null) return undefined;
	cur = cur[p];
  }
  return cur;
}
Returns / End result:
Returns the resolved value (or undefined if a segment is missing).
 
Method: setPath(path: string, value: any): void
Description:
Sets a dot-delimited path from the Character root. Creates intermediate objects if missing.
Implementation details (JavaScript pseudocode):
setPath(path, value) {
  const parts = path.split('.');
  let cur = this;
  for (let i = 0; i < parts.length - 1; i++) {
	const p = parts[i];
	if (cur[p] == null || typeof cur[p] !== 'object') cur[p] = {};
	cur = cur[p];
  }
  cur[parts[parts.length - 1]] = value;
}
Returns / End result:
Returns void.

Adds an item to Inventory.Items and applies FeaturesOnObtain (triggered as onObtain).
Implementation details (JavaScript pseudocode):
addItem(item) {
  this.Inventory.Items.push(item);
  for (const f of (item.FeaturesOnObtain || [])) {
	this.applyFeature('onObtain', item.Id, f);
  }
}
Returns / End result:

Method: removeItem(itemId: string): void
Description:
Removes an item from Inventory.Items and reverses all feature instances applied from that sourceId (obtained/equipped/etc.).
Implementation details (JavaScript pseudocode):
removeItem(itemId) {
  this.reverseAllFeaturesFromSource(itemId);
  this.Inventory.Items = this.Inventory.Items.filter(i => i.Id !== itemId);
}
Returns / End result:
Returns void. Item removed; all source-linked feature instances reversed.
 
Method: equipItem(itemId: string): void
Description:
Marks an item equipped and applies FeaturesOnEquip (triggered as onEquip).
Implementation details (JavaScript pseudocode):
equipItem(itemId) {
  const item = this.findAnyItemById(itemId);
  if (!item) throw new Error('item not found');
  if (item.Equipped) return;
  item.Equipped = true;
  for (const f of (item.FeaturesOnEquip || [])) {
	this.applyFeature('onEquip', item.Id, f);
  }
}
Returns / End result:
Returns void. Item equipped; equip features applied.
 
Method: unequipItem(itemId: string): void
Description:
Reverses prior onEquip feature instances for this item and marks it unequipped. Optionally applies FeaturesOnUnequip (triggered as onUnequip).
Implementation details (JavaScript pseudocode):
unequipItem(itemId) {
  const item = this.findAnyItemById(itemId);
  if (!item) throw new Error('item not found');
  if (!item.Equipped) return;
  this.reverseFeaturesBySourceAndTrigger(item.Id, 'onEquip');
  item.Equipped = false;
  for (const f of (item.FeaturesOnUnequip || [])) {
	this.applyFeature('onUnequip', item.Id, f);
  }
}
Returns / End result:
Returns void. Equip effects reversed; item is unequipped.

Method: computeStatMods(statsMap: object): object
Description:
Computes stat modifiers for STR/DEX/CON/WIS/INT/CHA using floor((stat - 10) / 2).
Implementation details (JavaScript pseudocode):
function computeStatMods(statsMap) {
  const modMap = {};
  for (const stat of ['STR','DEX','CON','WIS','INT','CHA']) {
	const v = statsMap[stat] ?? 0;
	modMap[stat] = Math.floor((v - 10) / 2);
  }
  return modMap;
}

Method: computeAttackMods(modMap: object, weaponArray: WeaponObject[], profBonus: int): WeaponObject[]
Description:
For each weapon, chooses the best attack modifier across its PrimaryStats. Adds proficiency bonus if weapon.Proficient, and adds weapon.AttackBonus. Stores result into weapon.attackMod.
Implementation details (JavaScript pseudocode):
function computeAttackMods(modMap, weaponArray, profBonus) {
  for (const weapon of weaponArray) {
	let wMod = 0;
	for (const primaryStat of (weapon.PrimaryStats || [])) {
  	let candidate = modMap[primaryStat] ?? 0;
  	if (weapon.Proficient) candidate += profBonus;
  	candidate += (weapon.AttackBonus ?? 0);
  	wMod = Math.max(wMod, candidate);
	}
	weapon.attackMod = wMod;
  }
  return weaponArray;
}
Returns / End result:
Returns weaponArray with updated weapon.attackMod values.
 
Method: computeSaves(sMods: object, primaryClassSavingThrows: stats[], profBonus: int): object
Description:
Computes saves: base save equals sMods[stat]. If stat is in primaryClassSavingThrows, adds ProficiencyBonus. Stores results in a save map.
Implementation details (JavaScript pseudocode):
function computeSaves(sMods, primaryClassSavingThrows, profBonus) {
  const saves = {};
  for (const stat of ['STR','DEX','CON','WIS','INT','CHA']) {
	let v = sMods[stat] ?? 0;
	if (primaryClassSavingThrows?.includes(stat)) v += profBonus;
	saves[stat] = v;
  }
  return saves;
}

Returns Map<stat → int>.
